# RISC-V CPU FPGA Project

This repository contains a Vivado 2019.2 project for a small RV32I-style pipelined CPU running from a True Dual Port BRAM.

## Status

- Vivado project: `RISCV_FPGA.xpr`
- Device: `xc7a100tfgg676-2`
- Synthesis top: `riscv_mcu_tdpram_top`
- Simulation top: `riscv_mcu_top_tb`
- CPU core: `RISCV_FPGA.srcs/sources_1/new/riscv_cpu.v`
- Program image: `test7_tdp.coe`
- BRAM IP: `RISCV_FPGA.srcs/sources_1/ip/tdp_bram/tdp_bram.xci`
- Clock IP: `RISCV_FPGA.srcs/sources_1/ip/clk_wiz_sys/clk_wiz_sys.xci`

The current timing constraint is 120 MHz:

```tcl
create_clock -name clk -period 8.333 [get_ports CLK_FPGA]
```

Latest routed result in Vivado 2019.2:

```text
WNS = +0.408 ns
TNS = 0.000 ns
Failing endpoints = 0
Hold WHS = +0.166 ns
```

## CPU Features

- IF / ID / EX / MEM / WB pipeline
- ID-stage forwarding select predecode
- EX-stage forwarding data mux
- load-use stall
- branch / jal / jalr flush
- byte, halfword, and word load/store alignment
- dedicated load/store address adder for better BRAM timing
- True Dual Port BRAM instruction/data access

## Quick Start

1. Clone the repository.

   ```powershell
   git clone https://github.com/jerry1163/RISCV_CPU.git
   cd RISCV_CPU
   ```

2. Open the Vivado project.

   ```text
   RISCV_FPGA.xpr
   ```

3. If Vivado says IP output products are missing or stale, regenerate them.

   In the Vivado GUI, use `Reports` or the IP status window and choose `Generate Output Products`.

4. Run synthesis and implementation.

   The project is currently constrained at 120 MHz. The latest checked implementation meets timing with positive slack.

## Recreate the Project

If the checked-in `.xpr` keeps stale machine paths after cloning, recreate it from relative paths:

```tcl
cd <repo-root>
source scripts/recreate_project.tcl
```

Then open the regenerated `RISCV_FPGA.xpr`.

## Simulation

In Vivado, run behavioral simulation with:

```text
Run Simulation -> Run Behavioral Simulation
```

The simulation top is:

```text
riscv_mcu_top_tb
```

The existing `test7_tdp.coe` program should produce representative final register values such as:

```text
x01=87654321
x02=04040404
x06=00000028
x07=00000030
x25=87654088
x26=ffffffef
x27=ffffcdef
x28=000000ef
x29=0000cdef
x30=00000118
x31=00000140
```

## BRAM Initialization

`tdp_bram` is generated by Vivado Block Memory Generator. The IP points to:

```text
test7_tdp.coe
```

The generated simulation memory file is:

```text
RISCV_FPGA.srcs/sources_1/ip/tdp_bram/tdp_bram.mif
```

If the IP is regenerated, verify that the BRAM contents still match `test7_tdp.coe`.

## Notes Before Bitstream

The repository is ready for synthesis, implementation, and simulation. Before generating a board bitstream, add board-specific `LOC` and `IOSTANDARD` constraints for `CLK_FPGA` and `PA[31:0]`.

Vivado may also warn that `PA[31:0]` has no input/output delay constraints. Those are external interface constraints and should be filled in according to the target board and peripheral timing.
