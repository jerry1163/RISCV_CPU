# RISC-V CPU FPGA Project

This repository contains a Vivado 2019.2 project for a small RV32I-style pipelined CPU running from a True Dual Port BRAM.

## Status

- Vivado project: `RISCV_FPGA.xpr`
- Device: **xc7k325tffg900-2** (Kintex-7, upgraded from Artix-7)
- Synthesis top: `riscv_mcu_tdpram_top`
- Simulation top: `riscv_mcu_top_tb`
- CPU core: `RISCV_FPGA.srcs/sources_1/new/riscv_cpu.v`
- Program image: `test7_tdp.coe`
- BRAM IP: `RISCV_FPGA.srcs/sources_1/ip/tdp_bram/tdp_bram.xci`
- Clock IP: `RISCV_FPGA.srcs/sources_1/ip/clk_wiz_sys/clk_wiz_sys.xci`

**Current timing constraint: 210 MHz on Kintex-7.**

The design previously ran at 150 MHz on Artix-7 (xc7a100tfgg676-2) with WNS = -0.093 ns. After migrating to Kintex-7, the same RTL passes timing at 210 MHz with positive slack.

### In-Progress: RV32M Extension

- Multiplier IP: `RISCV_FPGA.srcs/sources_1/ip/mult_32x32/mult_32x32.xci` (DSP48-based, 1-cycle)
- Divider IP: `RISCV_FPGA.srcs/sources_1/ip/div_gen_0/div_gen_0.xci` (High Radix, pipelined)
- Integration into the CPU pipeline is pending.

## CPU Features

- IF / ID / EX / MEM / WB 5-stage pipeline
- Full forwarding (EX/MEM and MEM/WB bypass)
- ID-stage forwarding select predecode
- load-use stall (1 cycle penalty)
- branch / jal / jalr flush (3-cycle penalty)
- byte, halfword, and word load/store alignment
- dedicated load/store address adder
- True Dual Port BRAM for independent instruction/data access

## Quick Start

1. Clone the repository.

   ```powershell
   git clone https://github.com/jerry1163/RISCV_CPU.git
   cd RISCV_CPU
   ```

2. Open the Vivado project (2019.2).

   ```
   RISCV_FPGA.xpr
   ```

3. If Vivado reports stale IP output products:

   In the Vivado GUI, use `Reports > Report IP Status` and choose `Generate Output Products` for any stale IPs.

4. Run synthesis and implementation.

   The project targets Kintex-7 (xc7k325tffg900-2) at 210 MHz via `synth_2` / `impl_2` runs. To switch back to Artix-7, use `synth_1` / `impl_1`.

## Recreate the Project

If the checked-in `.xpr` contains stale machine paths, recreate from relative paths:

```tcl
cd <repo-root>
source scripts/recreate_project.tcl
```

## Simulation

In Vivado, run behavioral simulation with:

```
Run Simulation > Run Behavioral Simulation
```

The simulation top is `riscv_mcu_top_tb`.

The `test7_tdp.coe` program produces representative final register values:

```
x01=87654321   x02=04040404   x06=00000028   x07=00000030
x25=87654088   x26=ffffffef   x27=ffffcdef   x28=000000ef
x29=0000cdef   x30=00000118   x31=00000140
```

## BRAM Initialization

`tdp_bram` is generated by Vivado Block Memory Generator. The IP references `test7_tdp.coe`. If the IP is regenerated, verify the BRAM contents still match.

## Notes Before Bitstream

Before generating a board bitstream, add board-specific `LOC` and `IOSTANDARD` constraints for `CLK_FPGA` and `PA[31:0]`. Vivado may also warn that `PA[31:0]` has no I/O delay constraints — these should be filled in according to the target board and peripheral timing.
